What are Operators?

What are Operators?

Operators are special symbols or keywords that perform operations on operands (values or variables). They are the fundamental building blocks of expressions in programming and mathematics.

üî¢
Arithmetic Operators
Perform mathematical calculations like addition, subtraction, multiplication, and division.
üîç
Comparison Operators
Compare values and return true or false based on the relationship between operands.
‚ö°
Logical Operators
Combine or modify boolean values to create complex logical expressions.
Key Concept: Operators work with operands to produce results. For example, in "5 + 3", "+" is the operator and "5" and "3" are the operands, producing the result "8".

Basic Arithmetic Operators

User Authentication

username = "admin" password = "secret123" is_logged_in = false // Check login credentials valid_user = (username == "admin") && (password == "secret123") can_access = valid_user && is_logged_in // Alternative access (admin OR premium user) is_premium = true has_access = (username == "admin") || is_premium

Game Logic

player_level = 15 has_key = true boss_defeated = false // Can enter special area? can_enter = (player_level >= 10) && has_key // Game completion game_won = boss_defeated && (player_level >= 20) // Show hint (new player OR struggling) show_hint = (player_level < 5) || (!boss_defeated && player_level < 15)
Short-Circuit Evaluation: Logical operators use short-circuit evaluation. In AND (&&), if the first condition is false, the second isn't evaluated. In OR (||), if the first condition is true, the second isn't evaluated.
Operator Precedence
Order of Operations

Precedence Hierarchy

From Highest to Lowest Priority:

  1. Parentheses: ( )
  2. Exponentiation: **
  3. Unary operators: !, -, +
  4. Multiplication, Division, Modulus: *, /, %
  5. Addition, Subtraction: +, -
  6. Comparison: <, <=, >, >=
  7. Equality: ==, !=
  8. Logical AND: &&
  9. Logical OR: ||

Complex Expression Examples

// Expression: 3 + 4 * 2 > 10 && 5 < 8 // Step 1: 4 * 2 = 8 (Multiplication) // Step 2: 3 + 8 = 11 (Addition) // Step 3: 11 > 10 = true (Comparison) // Step 4: 5 < 8 = true (Comparison) // Step 5: true && true = true (Logical AND) // Result: true // Expression: !(5 > 3) || (2 * 3 == 6) // Step 1: 5 > 3 = true (Comparison) // Step 2: !(true) = false (Logical NOT) // Step 3: 2 * 3 = 6 (Multiplication) // Step 4: 6 == 6 = true (Equality) // Step 5: false || true = true (Logical OR) // Result: true
Best Practice: Use parentheses to make your intentions clear, even when not strictly necessary. This improves code readability and prevents errors.

üîó Ternary Operator

Pitfalls to Avoid

Frequent Programming Errors

‚ùå Assignment vs Comparison

// WRONG - Assignment instead of comparison if (x = 5) { // This assigns 5 to x, doesn't compare! // Code here } // CORRECT - Comparison if (x == 5) { // This compares x with 5 // Code here }

‚ùå Floating Point Precision

// PROBLEMATIC - Direct floating point comparison a = 0.1 + 0.2 b = 0.3 if (a == b) { // May not work due to precision errors! // This might not execute } // BETTER - Use tolerance for comparison tolerance = 0.0001 if (Math.abs(a - b) < tolerance) { // This is more reliable }

‚ùå Operator Precedence Confusion

// UNCLEAR - Hard to read, prone to errors result = 5 + 3 * 2 > 10 && 4 < 6 || false // CLEAR - Use parentheses for clarity result = ((5 + (3 * 2)) > 10) && (4 < 6) || false // ((5 + 6) > 10) && (true) || false // (11 > 10) && true || false // true && true || false // true || false // true

‚ùå Logical Operator Misuse

// WRONG - Using & instead of && if (condition1 & condition2) { // Bitwise AND, not logical! // May not work as expected } // CORRECT - Logical AND if (condition1 && condition2) { // This is what you usually want }
