What is Type Casting?

Definition

Type Casting is the process of converting a variable from one data type to another data type.

Why is Type Casting Important?

  • Data Compatibility: Ensures different data types can work together
  • Memory Optimization: Converts to appropriate data types to save memory
  • Mathematical Operations: Enables calculations between different numeric types
  • User Input Processing: Converts string input to required data types

Simple Example:

int number = 10; double decimal = (double)number; // Casting int to double // Result: decimal = 10.0

Types of Type Casting

1. Implicit Casting (Automatic)

  • Performed automatically by the compiler
  • No data loss occurs
  • Smaller to larger data types
  • Also called "Widening Conversion"
int a = 10; double b = a; // Implicit casting // int → double

2. Explicit Casting (Manual)

  • Performed manually by programmer
  • May result in data loss
  • Larger to smaller data types
  • Also called "Narrowing Conversion"
double x = 10.75; int y = (int)x; // Explicit casting // Result: y = 10 (decimal lost)

Implicit Type Casting

Hierarchy of Data Types (Small to Large):

byte → short → int → long → float → double char → int

Examples:

// C# Examples byte b = 10; short s = b; // byte to short int i = s; // short to int long l = i; // int to long float f = l; // long to float double d = f; // float to double // Character to integer char c = 'A'; int ascii = c; // Result: ascii = 65
Note: Implicit casting is safe because no data is lost in the conversion process.

Explicit Type Casting

Syntax: (target_type) variable

Basic Examples:

// C# Examples double price = 99.99; int roundedPrice = (int)price; // Result: 99 float temperature = 36.7f; int temp = (int)temperature; // Result: 36 long bigNumber = 1000000L; int smallNumber = (int)bigNumber; // Potential data loss!
Warning: Explicit casting can result in data loss or overflow. Always ensure the target variable can hold the converted value.

String to Numeric Conversions:

string numberStr = "123"; int number = int.Parse(numberStr); // Method 1 int number2 = Convert.ToInt32(numberStr); // Method 2

Type Casting Methods

Parse Methods

int.Parse("123") double.Parse("45.67") bool.Parse("true") DateTime.Parse("2024-01-01")

Convert Class

Convert.ToInt32("123") Convert.ToDouble("45.67") Convert.ToBoolean("true") Convert.ToString(123)

TryParse Methods (Safe)

string input = "123"; int result; if (int.TryParse(input, out result)) { // Conversion successful Console.WriteLine(result); } else { // Conversion failed Console.WriteLine("Invalid input"); }
Best Practice: Use TryParse methods when dealing with user input to avoid exceptions.

User Input in Programming

What is User Input?

User input refers to data that a user provides to a program during its execution. This can include text, numbers, file selections, mouse clicks, or keyboard interactions.

Common Input Sources:

  • Console Input: Command-line text input
  • GUI Input: Text boxes, buttons, forms
  • File Input: Reading from files
  • Network Input: Data from web requests or APIs
  • Device Input: Sensors, cameras, microphones

Basic Console Input (C#):

Console.WriteLine("Enter your name:"); string name = Console.ReadLine(); Console.WriteLine($"Hello, {name}!");

Console Input Methods

C# Console Methods

// Read a line of text string line = Console.ReadLine(); // Read a single character int ch = Console.Read(); // Read a key press ConsoleKeyInfo key = Console.ReadKey();

Python Input Methods

# Read a line of text name = input("Enter name: ") # Read and convert to int age = int(input("Enter age: ")) # Read and convert to float salary = float(input("Salary: "))

Complete Example - Getting User Information:

// C# Example Console.Write("Enter your name: "); string name = Console.ReadLine(); Console.Write("Enter your age: "); string ageInput = Console.ReadLine(); int age = int.Parse(ageInput); Console.Write("Enter your salary: "); string salaryInput = Console.ReadLine(); double salary = double.Parse(salaryInput); Console.WriteLine($"Name: {name}, Age: {age}, Salary: ${salary:F2}");

Input Validation

Why Validate Input?

  • Prevent program crashes from invalid data
  • Ensure data integrity and correctness
  • Provide better user experience
  • Security - prevent malicious input

Validation Example with TryParse:

// Safe integer input with validation Console.Write("Enter your age: "); string input = Console.ReadLine(); int age; while (!int.TryParse(input, out age) || age < 0 || age > 150) { Console.WriteLine("Please enter a valid age (0-150):"); input = Console.ReadLine(); } Console.WriteLine($"Your age is: {age}");
Best Practice: Always validate user input before using it in your program logic.

Advanced Input Validation

Comprehensive Validation Function:

public static int GetValidInteger(string prompt, int min, int max) { int result; string input; do { Console.Write(prompt); input = Console.ReadLine(); if (!int.TryParse(input, out result)) { Console.WriteLine("Error: Please enter a valid integer."); } else if (result < min || result > max) { Console.WriteLine($"Error: Please enter a number between {min} and {max}."); } else { return result; // Valid input } } while (true); } // Usage int age = GetValidInteger("Enter your age (1-120): ", 1, 120);

Error Handling with User Input

Try-Catch Approach:

try { Console.Write("Enter a number: "); string input = Console.ReadLine(); int number = int.Parse(input); Console.WriteLine($"You entered: {number}"); Console.WriteLine($"Double the number: {number * 2}"); } catch (FormatException) { Console.WriteLine("Error: Invalid number format!"); } catch (OverflowException) { Console.WriteLine("Error: Number is too large!"); } catch (Exception ex) { Console.WriteLine($"Unexpected error: {ex.Message}"); }
Note: While try-catch works, TryParse methods are generally preferred for input validation as they don't throw exceptions.

Practical Examples

Example 1: Calculator with Input Validation

public static void SimpleCalculator() { Console.WriteLine("=== Simple Calculator ==="); // Get first number double num1 = GetValidDouble("Enter first number: "); // Get operation Console.Write("Enter operation (+, -, *, /): "); char operation = Console.ReadKey().KeyChar; Console.WriteLine(); // Get second number double num2 = GetValidDouble("Enter second number: "); // Perform calculation double result = 0; switch (operation) { case '+': result = num1 + num2; break; case '-': result = num1 - num2; break; case '*': result = num1 * num2; break; case '/': if (num2 != 0) result = num1 / num2; else { Console.WriteLine("Error: Division by zero!"); return; } break; default: Console.WriteLine("Invalid operation!"); return; } Console.WriteLine($"Result: {num1} {operation} {num2} = {result}"); }

Best Practices

Type Casting Best Practices

  • Use TryParse for safe conversions
  • Check for data loss in explicit casting
  • Use appropriate data types for your needs
  • Handle overflow and underflow conditions
  • Document any potential data loss

User Input Best Practices

  • Always validate user input
  • Provide clear error messages
  • Give users multiple chances to correct input
  • Sanitize input to prevent security issues
  • Use appropriate input methods for data types

Golden Rules

  • Never trust user input - Always validate and sanitize
  • Fail gracefully - Provide meaningful error messages
  • Be consistent - Use the same validation patterns throughout your application
  • Plan for edge cases - Consider what could go wrong
